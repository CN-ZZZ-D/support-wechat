# 方案

## 技术选型

| 组件 | 技术 | 说明 |
|------|------|------|
| 桥接服务 | **Python + FastAPI** | 高性能异步 Web 框架，处理微信 XML 协议转换与异步任务调度 |
| AI 平台 | **Dify** | 可视化 AI 应用编排平台，提供标准化 `/v1/chat-messages` API |
| 消息通道 | **微信公众号 API** | 使用服务号客服消息接口 (`/cgi-bin/message/custom/send`) 突破 5 秒响应限制 |
| 辅助库 | `requests`, `xmltodict`, `asyncio` | HTTP 请求、XML 解析、后台异步执行 |
| 部署环境 | **Uvicorn + Nginx** | ASGI 服务器 + HTTPS 反向代理（微信强制要求） |

## 架构设计

```
微信用户 ←→ 微信公众号服务器 ←→ 桥接服务 (FastAPI) ←→ Dify API ←→ 本地/云端大模型
```

- **微信端**：用户发送文本消息至公众号，微信服务器将 XML 格式数据 POST 到桥接服务
- **桥接层**：接收消息后**立即返回 `success`** 避免超时，同时创建异步任务调用 Dify
- **AI 层**：Dify 接收请求，调用底层模型生成回复，返回完整 JSON 结果
- **回调层**：桥接服务获取 AI 回复后，通过微信客服接口异步推送给用户，实现“打字聊天”体验

## 实现步骤

### Step 1：准备微信公众号
1. 注册并认证**微信服务号**（订阅号无客服消息权限）
2. 进入 `设置与开发` → `基本配置`，获取 `AppID`、`AppSecret`，设置 `Token` 与 `EncodingAESKey`
3. 完成域名 ICP 备案并配置 HTTPS 证书

### Step 2：部署与配置 Dify
1. 本地或服务器部署 Dify（推荐 Docker）
2. 创建 Chat App 并调试 Prompt，点击 `发布`
3. 进入 `API 访问`，复制 `API Base URL`（如 `http://localhost/v1`）与 `API Key`

### Step 3：编写桥接服务
1. 创建 Python 虚拟环境，安装依赖：`pip install fastapi uvicorn requests xmltodict python-dotenv`
2. 编写 `main.py`，实现：
   - GET `/wechat`：微信服务器 URL 验证（SHA1 签名校验）
   - POST `/wechat`：接收 XML 消息，提取文本，**异步调用 Dify**
   - `process_and_reply_wechat()`：调用 Dify API → 获取 `conversation_id` 保持上下文 → 通过客服接口推送回复
3. 配置 `.env` 文件存放敏感凭证

### Step 4：配置 HTTPS 与反向代理
1. 使用 Nginx 将 `https://你的域名/wechat` 反向代理至 `http://127.0.0.1:8000/wechat`
2. 配置 SSL 证书（Let's Encrypt 或云厂商免费证书）
3. 测试 HTTPS 连通性：`curl -I https://你的域名/wechat`

### Step 5：微信公众号后台对接
1. 登录微信公众平台 → `设置与开发` → `基本配置` → `服务器配置`
2. 填写 URL：`https://你的域名/wechat`，Token 与代码中一致
3. 点击 `提交`，验证通过后 `启用` 服务器配置

### Step 6：测试与上线
1. 关注该公众号，发送文字消息
2. 查看服务器日志：`📥 收到消息` → `🤖 调用 Dify` → `📤 发送客服消息`
3. 监控 48 小时互动限制与 Access Token 刷新状态，稳定运行

## 各部件说明

### 桥接服务 (FastAPI)
- 核心中枢，负责协议转换（XML ↔ JSON）、异步防超时、会话记忆管理
- 使用 `asyncio.create_task()` 确保微信 5 秒内收到 `success`，避免重复推送
- 内存缓存 `conversation_store` 实现多轮对话上下文保持

### Dify
- AI 能力提供方，管理 Prompt 模板、模型路由、对话历史
- 通过标准 REST API 对外输出，支持 `blocking` 模式一次性返回完整结果
- 提供可视化调试界面，方便优化 AI 回复质量

### 微信公众号 API
- 消息收发通道，使用**客服消息接口**绕过普通消息 5 秒超时限制
- 依赖 `access_token` 鉴权，需定时刷新并缓存
- 受微信官方规则限制：仅支持向 48 小时内主动互动过的用户推送消息

### Nginx / HTTPS
- 安全网关，满足微信服务器配置强制 HTTPS 要求
- 提供负载均衡、SSL 终止、请求日志记录
- 可配置限流与防刷策略保护桥接服务

## 注意事项

- **⏱ 5 秒超时规则**：微信要求接口必须在 5 秒内响应，必须采用“先返回 success，后异步回复”架构
- **🕒 48 小时互动限制**：客服接口仅支持向 48 小时内发过消息的用户推送，超时需引导用户重新触发
- ** Access Token 管理**：微信 Token 有效期 2 小时，需缓存并提前 5 分钟刷新，频繁请求会被限频
- ** HTTPS 强制要求**：微信服务器配置仅支持 `https://` 域名，且域名必须完成 ICP 备案
- **💾 会话持久化**：示例使用内存字典存储 `conversation_id`，重启会丢失上下文。生产环境建议替换为 Redis
- **🛡 内容安全**：建议接入微信内容安全 API (`/wxa/msg_sec_check`)，避免违规内容导致接口封禁
