# 编码要求

> 本文档定义 AI 改动代码时的规范，用户主动提出时 AI 参考。

## 颗粒化规范

- 每次改动尽量小，聚焦单一功能点
- 避免一次性大规模重构
- 每完成一个小步骤，验证后再继续

## 低 Token 规范

- 优先使用编辑（Edit）而非重写（Write），减少 token 消耗
- 复用已有代码模式和工具函数
- 避免不必要的注释和空行

## 代码风格

- 遵循项目已有的代码风格
- Python 项目遵循 PEP 8
- JavaScript/TypeScript 项目遵循项目内的 lint 配置

## 提交规范

- 每次改动后运行相关测试确保不破坏现有功能
- 记录改动内容到对应 day 的 record 中
