# 项目说明

## 目录结构

```
app/          各功能部件（backend / frontend）
docs/         文档（想法、环境、方案）
test/         测试用例（按步骤拆分）
records/      AI 运行记录（分阶段管理）
skills/       规范文件（编码要求、前端设计等）
days/         用户自管理（随笔、日记、临时文件）
```

## 工作流程

1. **阶段一**：用户填写 `docs/想法.md` 和 `docs/环境.md`
2. **阶段二**：AI 生成 `docs/方案.md`，与用户商榷确认
3. **阶段三**：初始化 `app/` 及 `test/` 各部件，开始开发

## 新会话启动

每次新建会话时，请让 AI 先读取 `records/record.md` 以了解全局注意事项和当前进度。
